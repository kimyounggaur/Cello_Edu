# 🎻 첼로 운지·계이름 학습 웹앱 — 모바일 최적화 바이브코딩 프롬프트 (통합 마스터본)

> **목적** : AI 코딩 도구(Claude Code · Cursor · v0 · Bolt · Lovable · Windsurf · Replit Agent 등)에 그대로 붙여넣어, 첼로 운지법과 계이름 연주를 **가장 직관적이고 모바일 친화적으로** 배우는 웹앱을 단계적으로 완성하기 위한 설계서.
>
> **이 문서의 3대 핵심 원칙**
> 1. **정확성의 단일 출처** — 모든 음/주파수/운지는 §5의 `celloData.ts`에서만 계산한다. AI가 첼로 이론을 추측·하드코딩하지 못하게 막는 것이 이 문서의 가장 중요한 역할이다. (Claude 전략)
> 2. **모바일 퍼스트** — 한 손 엄지로 끝까지 조작 가능한 네이티브 앱 수준의 UX. 이 문서는 모바일 UI/UX(§3)에 가장 많은 분량을 할애한다.
> 3. **단계별 검증** — "전부 만들어줘"는 부정확하고 거대한 코드를 낳는다. **Phase 0 → 9**를 하나씩, 각 단계의 **수용 기준 ✅** 을 통과시키며 진행한다. (Claude/ChatGPT 전략)
>
> **앱 임시 이름** : **지판온(Jipan-On)** — "지판(fingerboard) + 음(音/sound)". 자유롭게 바꿔도 된다.
> **기본 언어** : 한국어 UI. **기본 조율** : C2-G2-D3-A3 (완전5도). **기본 계이름** : 고정도법(도=C).

## 0. 빠른 시작 (TL;DR)

1. **§5의 `celloData.ts`를 먼저 프로젝트에 넣는다.** (모든 음 정보의 기준 — 이걸 빼먹으면 AI가 음 이름을 지어낸다)
2. **§2 마스터 프롬프트**를 AI에게 한 번 붙여넣어 골격·계획을 확인받는다. (이때 `celloData.ts`도 함께 첨부하면 최상)
3. **§8 Phase 0 → 9를 한 번에 하나씩** 붙여넣고, 각 단계의 **수용 기준 ✅** 을 통과시킨 뒤 다음으로 넘어간다.
4. 막히거나 음이 틀리면 **§9 정확성 가드레일**을, UI가 어색하면 **§10 개선 프롬프트**를 다시 붙여넣어 교정한다.

> ⚠️ **한 번에 전부 시키지 말 것.** 단계별로, 검증하며 진행하는 것이 이 설계의 핵심이다.
> ⏱️ **시간이 없으면** §15의 "한 번에 붙여넣는 초압축 프롬프트"만 사용해도 동작하는 MVP가 나온다.

## 1. 제품 정의 — "가장 직관적인 첼로 학습 앱"의 기준

### 1.1 한 줄 정의
> 첼로 지판 위에서 *"이 계이름을 어느 줄·포지션·손가락으로 짚고, 어떤 소리가 나는가"* 를 **보고 · 듣고 · 눌러보고 · 퀴즈로 확인**하며 익히는 인터랙티브 모바일 학습 웹앱.

### 1.2 타깃 사용자
- 첼로를 처음 배우는 입문~초중급 독학자 및 학생
- 악보의 음표와 실제 지판 위치가 연결되지 않는 학습자
- 계이름은 아는데 첼로에서 어디를 눌러야 하는지 모르는 학습자
- 레슨 사이 혼자 복습하려는 학생, 보조 교구가 필요한 교사 (한국어 UI 기본)

### 1.3 사용자가 끝내 할 수 있어야 하는 일 (Job To Be Done)
1. 임의의 음(계이름/음이름)을 보면 → 지판에서 짚을 자리를 **즉시 떠올린다**.
2. ½~IV 포지션의 **손가락 모양(프레임)** 을 손에 익힌다.
3. 계이름(도레미)·음이름(다라마/CDEFGAB)·실제 음높이를 **하나로 연결**한다.

### 1.4 초보자가 반드시 해결해야 하는 혼란 (이 표가 기능의 근거)

| 초보자의 혼란 | 앱이 제공해야 할 해결 |
|---|---|
| C/G/D/A 현 위치를 헷갈림 | 현 이름을 항상 고정 표시 + 현별 색 + 텍스트 라벨 병행 |
| 1번 손가락이 어느 음인지 모름 | "A현 1번 = B = 시"처럼 현+손가락+음이름+계이름을 한 카드(바텀시트)로 표시 |
| 같은 계이름이 여러 위치라 헷갈림 | "초급 추천 위치"와 "대체 위치"를 분리 표시 |
| 음이름 C와 계이름 도의 관계 혼동 | 고정도법 기본값(C=도) + 음이름/계이름 동시 표기 |
| 악보와 지판이 연결되지 않음 | 마커 탭 → 소리 + 모든 이름을 즉시 연결 |
| 소리를 안 듣고 외움 | 모든 학습 요소에 "탭하면 0.1초 내 재생" |
| **바이올린처럼 1포지션이 온음일 거라 오해** | **닫힌 자세는 손가락 1·2·3·4가 연속 반음(0·1·3·4 패턴)임을 시각·문구로 강조** |

### 1.5 성공 기준 (북극성 지표)
기능 개수가 아니라, 학습자가 다음 문장을 말할 수 있는지로 판단한다.

> "A현 1번 손가락을 누르면 B, 계이름 시가 나고, 이 소리가 바로 그 음이구나."

**최소 성공 시나리오** : 사용자가 `A현 개방현`을 탭 → A3/라 표시·재생 → `A현 1번`을 탭 → B3/시 표시·재생 → 계이름 멜로디 `라 시 도 레`를 보고 지판에서 순서대로 탭 → 앱이 구체적으로 칭찬·교정 → "현·손가락·계이름·소리가 연결됐다"고 느낀다.

### 1.6 설계 5대 원칙 (AI는 모든 화면에서 이 원칙을 지킨다)
1. **직관성** — 글보다 **지판 그림**이 먼저. 색·위치·애니메이션으로 의미 전달. 설명 문장은 최소화.
2. **즉각 피드백** — 누르면 **0.1초 안에 소리 + 음이름**이 반응.
3. **정확성** — 모든 음/운지는 `celloData.ts`에서만. 하드코딩·추측 금지.
4. **모바일 퍼스트** — 한 손 엄지로 조작 가능한 큰 터치 타깃. 하단 내비·바텀시트 등 네이티브 패턴.
5. **접근성** — 키보드 조작, ARIA, 색맹 안전(색+모양+텍스트 병행), `prefers-reduced-motion` 존중.

## 2. 마스터 프롬프트 (맨 처음 1회 붙여넣기)

> 아래 코드블록 전체를 복사해 AI 코딩 도구에 **맨 처음 한 번** 붙여넣으세요. (§5의 `celloData.ts` 내용도 함께 첨부하면 가장 좋습니다.)

```text
너는 시니어 프론트엔드 엔지니어이자 음악 교육 UX 디자이너이며, 첼로 기초 이론을 이해하는 교육 앱 설계자다.
"지판온(Jipan-On)"이라는 첼로 운지·계이름 학습 모바일 웹앱을 나와 함께 단계적으로 만든다.

[제품]
- 첼로 지판 위에서 "이 계이름을 어느 줄·포지션·손가락으로 짚고 어떤 소리가 나는지"를
  보고·듣고·눌러보고·퀴즈로 확인하며 익히는 인터랙티브 학습 앱.
- 타깃: 첼로 입문~초중급 한국어 사용자. UI 기본 언어는 한국어.
- 1순위 환경은 "모바일 웹"이다. 데스크톱은 모바일 레이아웃을 확장해 대응한다.

[기술 스택 — 정확히 이대로]
- Vite + React 18 + TypeScript
- Tailwind CSS (디자인 토큰은 CSS 변수 + tailwind theme 확장)
- 오디오: Tone.js 합성음(신디사이저). 저작권 있는 샘플 사용 금지.
- 전역 상태: zustand (설정/진도만. persist 미들웨어로 localStorage 저장)
- 애니메이션: framer-motion (가볍게, 과하지 않게)
- 지판/마커는 인라인 SVG로 그린다 (확대해도 깨지지 않게)
- 백엔드 없음. 정적 배포(Vercel/Netlify/GitHub Pages) + PWA(설치 가능) 기준.

[가장 중요한 규칙 — 정확성]
- 첨부된 src/lib/celloData.ts 가 "정확성의 단일 출처(Single Source of Truth)"다.
- 음이름·주파수·포지션·운지 프레임은 절대 추측하거나 새로 하드코딩하지 말고,
  반드시 celloData.ts의 함수/데이터에서 계산해서 사용한다.
- 첼로 이론을 임의로 바꾸지 마라. 의심되면 멈추고 나에게 물어라.
- 특히: 닫힌 자세에서 손가락 1·2·3·4는 연속 "반음"이다(바이올린식 온음 기본과 다름).

[모바일 UI/UX 규칙 — 반드시 준수]
- 하단 탭바(스티키, safe-area 대응) + 화면 본문 + 하단 미니 컨트롤(재생/음소거/라벨 토글) 구조.
- 모든 터치 타깃 ≥ 48px, 인접 타깃 간 간격 ≥ 8px. 빽빽한 지판 마커는 보이지 않는 큰 히트영역 사용.
- 음 상세 정보는 화면을 가리지 않는 "바텀시트(drag-to-dismiss)"로 표시한다.
- 지판은 세로(portrait) 우선, 화면 폭을 꽉 채운다. 필요 시 가로 스크롤 허용(touch-action으로 페이지 스크롤과 충돌 방지).
- 본문 글꼴 ≥ 16px(iOS 입력 시 자동 확대 방지). 음 라벨은 더 크게.
- 탭 반응 100ms 이내 시각 피드백. touch-action: manipulation으로 300ms 지연 제거.
- (지원 시) navigator.vibrate로 정답/오답 햅틱(설정으로 끌 수 있게).
- 호버에 의존하지 말 것. 모든 기능은 탭으로 동작하고 호버는 보조일 뿐.
- env(safe-area-inset-*)로 노치/홈 인디케이터 영역 보호.

[작업 방식]
- 한 번에 전부 만들지 말고, 내가 "Phase N 시작"이라고 하면 그 단계만 구현한다.
- 큰 구조 변경(라우팅/상태/폴더 구조)이 필요하면 먼저 제안하고 내 확인을 받는다.
- 컴포넌트는 작게 쪼개고, 타입을 엄격히 쓴다(any 금지).
- 각 단계 끝에는 "수용 기준"을 스스로 점검하고, 통과 여부를 보고한다.
- 데이터 로직(celloData.ts)에는 간단한 런타임 자가점검(selfTest)을 포함하고 dev 모드에서 콘솔에 PASS/FAIL을 찍는다.

[디자인 방향 — "흑단과 황동(ebony & brass)", 라이트/세피아 토글 제공]
- 첼로의 세계에서 길어올린 톤: 따뜻한 흑단 지판, 황동(brass) 포인트, 상아(ivory) 음표.
- 다크 기본(연습실/야간 친화, OLED 절전)이되, 라이트(세피아) 테마 토글을 반드시 제공.
- 4개 현은 고유 색으로 구분(낮은음=따뜻하게, 높은음=밝게)하되 색만으로 구분하지 말고
  항상 음이름/손가락 텍스트를 함께 표기.
- 디스플레이 글꼴: Noto Serif KR / 본문: Noto Sans KR. 음악 기호 ♯ ♭ ♮ ½ 정상 표기.
- 불필요한 장식은 배제하고 "정보 전달"에 집중(노션식 절제). 부드러운 탭 피드백.

[정보 구조 — 하단 탭(라우트)]
1) 지판(Fingerboard)  2) 찾기(Finder: 음↔자리)  3) 연습(Practice/퀴즈)
4) 음계(Scales)  5) 더보기(설정/포지션 트레이너/레퍼런스 차트)
(주: 모바일 하단 탭은 5개 이하 유지. 포지션 트레이너·레퍼런스·설정은 "더보기"로 묶는다.)

지금은 코드를 쓰지 말고, 위 내용을 요약 확인한 뒤
제안하는 폴더 구조와 Phase 0 계획만 먼저 보여줘.
```

## 3. 📱 모바일 UI/UX 설계 (이 앱의 차별점 — 가장 중요)

> 이 앱의 1순위 환경은 모바일이다. 아래 패턴들을 AI가 모든 화면에 일관되게 적용하도록 한다. 데스크톱은 이 레이아웃을 확장한다(모바일을 데스크톱으로 축소하지 않는다).

### 3.1 화면 골격 (모든 페이지 공통)
```text
┌─────────────────────────────┐
│        (상태바 safe-area)        │
│  지판온            [라벨▾] [🔆]  │ ← 슬림 헤더(스티키, 48px). 우측: 라벨/테마 토글
├─────────────────────────────┤
│                                 │
│         화면별 본문(스크롤)         │ ← 핵심 콘텐츠. 지판은 폭을 꽉 채움
│                                 │
├─────────────────────────────┤
│   ◀◀   ▶ 재생   🔊   도레미/CDE   │ ← 미니 컨트롤(스티키, 항상 엄지 닿는 곳)
├─────────────────────────────┤
│  지판  찾기  연습  음계  더보기     │ ← 하단 탭바(safe-area-inset-bottom 패딩)
└─────────────────────────────┘
```
- 헤더·미니컨트롤·탭바는 `position: sticky`. 본문만 스크롤.
- 하단 요소는 `padding-bottom: env(safe-area-inset-bottom)`으로 홈 인디케이터를 피한다.

### 3.2 엄지 도달 영역(Thumb Zone) 원칙
```text
┌─────────────┐
│  ▒▒ 어려움 ▒▒  │  ← 상단: 정보 표시용(읽기 전용 라벨, 제목)
│  ░░ 보통 ░░░░  │  ← 중앙: 지판(탭 위주)
│  ██ 쉬움 ████  │  ← 하단: 주요 액션 버튼 전부 배치(재생/정답확인/다음)
└─────────────┘
```
- **주요 액션(재생, 정답 확인, 다음 문제, 모드 전환)은 화면 하단 1/3에 배치.** 상단에는 파괴적/주요 버튼 금지.
- 한 손 조작 가정. 양손이 필요한 인터랙션 금지.

### 3.3 인터랙티브 지판 (모바일 핵심 화면)
- **세로 지판**: 연주자 시점, 왼→오른쪽 = C·G·D·A. 위=너트, 아래=브리지(아래로 갈수록 고음). 화면 폭을 꽉 채운다.
- **마커 탭 → 바텀시트 상승**: 화면 하단에서 음 상세 카드가 슬라이드업. 지판을 가리지 않게 화면의 35~45%만 차지. **아래로 스와이프(drag-to-dismiss)** 로 닫기. (모바일에서 별도 영역 카드보다 훨씬 자연스러움)
- **터치 타깃**: 마커 시각 크기는 작아도, 실제 탭 영역(투명 hit area)은 ≥ 48px. 인접 마커와 ≥ 8px 간격. 겹치면 가로 스크롤/줌으로 해소.
- **자연음 vs 반음 시각 구분**: 자연음 = 상아색 원 안에 음이름. 반음(♯/♭) = 작은 링. 색맹 안전을 위해 모양으로도 구분.
- **표시 토글(헤더의 라벨▾)**: 음이름 / 계이름 / 손가락 번호 / 포지션 로마숫자 / 모두 끄기. 토글은 즉시 전 화면 반영.
- **포지션 가이드 라인**(½·I·II·III·IV)과 1포지션 영역 살짝 강조.
- **(선택) 핀치 줌·가로 스크롤**: `touch-action: pan-x pinch-zoom`으로 지판 영역만 제스처, 세로 페이지 스크롤과 충돌 방지.

### 3.4 모바일 네이티브 인터랙션 디테일
- **탭 즉시 반응**: 누르는 순간(0.1초 내) 마커 눌림(scale 0.96) + 색 변화 + 소리. "탭→결과" 사이 스피너 절대 금지(로컬 연산이므로 즉시).
- **300ms 지연 제거**: 인터랙티브 요소에 `touch-action: manipulation`.
- **햅틱**(지원 시 + 설정 ON): 정답 = 짧게 1회, 오답 = 짧게 2회(`navigator.vibrate([30,40,30])`). iOS는 미지원일 수 있으니 시각 피드백을 항상 병행.
- **오답 피드백**: 부드러운 흔들림(shake) 애니메이션 + 정답 자리/이름 안내. **정답**: 초록 체크 마이크로 인터랙션. (`prefers-reduced-motion`이면 모션 대신 색/아이콘만)
- **입력 확대 방지**: 모든 텍스트 ≥ 16px(특히 입력 필드). iOS의 포커스 시 자동 줌 방지.
- **스크롤 잠금**: 바텀시트가 열리면 배경 스크롤 잠금(body scroll lock).
- **작은 닫기 버튼 지양**: 바텀시트는 스와이프 다운 + 큰 핸들바로 닫기.

### 3.5 반응형 브레이크포인트
- **~480px (모바일, 1순위)**: 위 단일 컬럼 레이아웃. 지판 세로 풀폭.
- **481~1024px (태블릿)**: 지판 + 사이드 정보 패널 2분할 가능. 바텀시트 → 사이드 패널 전환 허용.
- **1025px~ (데스크톱)**: 중앙 정렬 컨테이너(최대폭 제한) + 좌측 지판/우측 패널. 하단 탭 → 상단 탭으로 이동 가능.

### 3.6 성능 & PWA
- **60fps 유지**: 애니메이션은 transform/opacity만(레이아웃 변경 애니메이션 금지). 지판 SVG 리렌더 최소화(메모이제이션).
- **즉시 로딩 체감**: 스켈레톤/즉시 인터랙션. 폰트는 `font-display: swap`.
- **PWA**: `manifest.json`(이름/아이콘/테마색/standalone), 서비스워커로 오프라인 캐싱(정적 앱이므로 가능), "홈 화면에 추가" 안내. 스플래시/앱 아이콘 제공.
- **오프라인 동작**: 백엔드가 없으므로 첫 로드 후 오프라인에서도 학습 가능해야 한다.

### 3.7 접근성(모바일 포함)
- 모든 인터랙티브 요소: 키보드 포커스 가능 + ARIA 라벨(예: `aria-label="A현 3번 손가락, 미, E, 1포지션"`).
- 색맹 안전: 색 + 모양 + 텍스트 3중. 대비 WCAG AA 이상.
- `prefers-reduced-motion`: 모든 모션 비활성(색/아이콘 피드백으로 대체).
- 스크린리더로 지판 탐색 가능한 대체 구조(리스트/표) 제공 고려.

## 4. 기능 명세 (상세)

### 4.1 인터랙티브 지판 (Fingerboard) — 앱의 심장
- 세로 지판 SVG(§3.3). 각 줄 위 반음 간격으로 음 자리(오프셋 0=개방현 ~ 약 10).
- 탭/호버 시: 해당 음 **재생** + 바텀시트(모바일)/말풍선(데스크톱)에 `음이름 · 계이름 · 손가락(해당 포지션) · 포지션` 표시.
- 자연음/반음 시각 구분, 라벨 토글, 포지션 가이드 라인.

### 4.2 찾기 (Finder: 음 ↔ 운지)
- 미니 건반/음 버튼에서 **목표 음 선택** → 지판에서 **연주 가능한 모든 자리 하이라이트**(포지션별 손가락 옵션 포함).
- **추천 운지**(가장 낮은 포지션/개방현 우선) 별도 강조 + 한 번에 재생.
- 역방향: 지판 자리를 누르면 그 음의 모든 이름(음이름/계이름) 표시.

### 4.3 포지션 트레이너 (Positions, "더보기" 안)
- ½·I·II·III·IV 중 선택 → **네 현의 닫힌 프레임**(손가락 1·2·3·4)을 한눈에.
- **손가락 1→4 순차 애니메이션** + 오름/내림 재생.
- 프레임 개념 설명: *닫힌 자세는 이웃 손가락이 모두 반음(1~4가 단3도 폭)*.
- **익스텐션 토글**: 앞 벌림(1 고정, 2·3·4 반음 위 → 1–2 온음, 장3도 폭) / 뒤 벌림(1만 반음 아래).

### 4.4 계이름·음이름 시스템 (전역 설정 + 모든 라벨에 반영)
라벨 시스템 4종(설정에서 선택):
1. **영문 음이름** : C D E F G A B (반음은 ♯/♭ 선택)
2. **한글 음이름(고정)** : 다 라 마 바 사 가 나
3. **계이름 · 고정도(도=C)** : 도 레 미 파 솔 라 시
4. **계이름 · 이동도(조 선택)** : 으뜸음 기준 도레미 (조성 드롭다운)

- 올림(♯)/내림(♭) 표기 선택 토글. (예: C♯ = 다♯ = 도♯)
- 매핑은 §6 표와 `celloData.ts`의 `displayName()`을 따른다.

### 4.5 오디오 엔진
- Tone.js 합성음(예: 삼각파+필터+엔벨로프, 첼로 느낌). 정확한 주파수는 `freqAt()` 사용.
- 사용자 제스처(첫 탭) 후 `Tone.start()`로 오디오 컨텍스트 활성화.
- 짧은 attack/release 엔벨로프로 클릭 노이즈 제거. 단음/멜로디 재생 구분. 음 잇기 시 이전 음 부드럽게 fade out. 음소거/볼륨.
- 나중에 실제 첼로 샘플을 붙일 수 있도록 `AudioEngine` 인터페이스를 추상화.

### 4.6 연습·퀴즈 (Practice)
- **모드 A (음 → 자리)**: 음(계이름/음이름/오선)을 제시 → 지판에서 정답 자리 탭 → 채점.
- **모드 B (자리 → 음)**: 지판 한 자리 하이라이트 → 음이름 4지선다.
- **모드 C (포지션 드릴)**: "이 포지션에서 이 음은 몇 번 손가락?" 맞히기.
- 난이도: 현 부분집합 / 포지션 부분집합 / 자연음만 vs 반음 포함.
- **연속 정답(streak)·정확도(%)·세션 요약**. 오답은 정답 자리·이름을 보여줌(아래 §11 피드백 문구 사용).
- 최근 틀린 음은 더 자주 출제. 같은 유형 3회 이상 연속 금지. 정답률 85%↑이면 다음 난이도 제안.

### 4.7 음계 & 간단한 곡 (Scales)
- **1옥타브 음계**(다장조/사장조/라장조 = 1포지션 친화) 운지 하이라이트 + 단계별 재생 + 메트로놈/템포.
- "따라 연주" 모드(다음에 짚을 자리 안내).
- 곡은 **퍼블릭 도메인/전통 멜로디만**(예: 작은별/반짝반짝). 저작권 있는 악보·멜로디 사용 금지.

### 4.8 레퍼런스 차트 (Reference, "더보기" 안)
- ½~IV 포지션 **운지표 전체**(현×포지션×손가락→음) + 조율표를 한 화면 참고용으로. (모바일에서는 가로 스크롤 표)

### 4.9 진도·설정·저장
- 설정: 라벨 시스템, ♯/♭, 음소거/볼륨, 템포, 모션 줄이기, 햅틱 ON/OFF, 다크/세피아(기본 다크).
- 진도: 퀴즈 정확도·streak·최근 학습 포지션을 localStorage에 저장. 초기화 버튼. 개인정보 수집 없음(로컬 저장만).

## 5. 음악 데이터 — `src/lib/celloData.ts` (정확성의 단일 출처, 검증 완료)

> 아래 파일을 **그대로** 프로젝트에 넣고, 값을 임의로 바꾸지 말 것. 주파수·포지션·익스텐션·음계는 표준 운지표(C2-G2-D3-A3 · 12평균율 · ½~IV 폐쇄형)에 대조해 검증되었다.

```ts
// src/lib/celloData.ts
// 첼로 운지/음이름 데이터 — 정확성의 단일 출처(Single Source of Truth)
// 이 파일의 값/공식은 표준 운지표에 대조 검증되었다. 임의로 바꾸지 말 것.

export type StringId = 'C' | 'G' | 'D' | 'A';
export type PositionId = 'half' | 'p1' | 'p2' | 'p3' | 'p4';
export type Finger = 1 | 2 | 3 | 4;

export interface CelloString {
  id: StringId;
  pc: number;     // 개방현 음의 피치 클래스(0=C..11=B)
  midi: number;   // 개방현 MIDI 번호
  sci: string;    // 과학적 표기 예:'C2'
  hz: number;     // 개방현 주파수
  color: string;  // UI 색
  gauge: number;  // 줄 두께(시각)
  order: number;  // 연주자 시점 왼→오른쪽 0=C..3=A
}

// 연주자 시점 왼→오른쪽: C, G, D, A (낮은음→높은음)
export const STRINGS: CelloString[] = [
  { id: 'C', pc: 0, midi: 36, sci: 'C2', hz: 65.41,  color: '#C9A36A', gauge: 6.0, order: 0 },
  { id: 'G', pc: 7, midi: 43, sci: 'G2', hz: 98.00,  color: '#CBB07A', gauge: 4.6, order: 1 },
  { id: 'D', pc: 2, midi: 50, sci: 'D3', hz: 146.83, color: '#D6CDBA', gauge: 3.3, order: 2 },
  { id: 'A', pc: 9, midi: 57, sci: 'A3', hz: 220.00, color: '#E0D8C6', gauge: 2.3, order: 3 },
];
export const stringById = (id: StringId) => STRINGS.find(s => s.id === id)!;

// 포지션: 1번 손가락이 개방현보다 몇 반음 위인지.
// 닫힌 자세에서 손가락 1·2·3·4는 연속 반음 → 1~4가 단3도(3반음) 폭.
// (참고: 표준 운지표의 II는 "낮은 2포지션(+3)", III는 "낮은 3포지션(+5)" 형을 채택.
//  "높은 2포지션(+4)", "높은 3포지션(+6)"은 레퍼런스 차트용 확장으로 §부록 참고.)
export interface Position { id: PositionId; label: string; romanKo: string; firstFingerSemitone: number; }
export const POSITIONS: Position[] = [
  { id: 'half', label: '½',   romanKo: '하프', firstFingerSemitone: 1 },
  { id: 'p1',   label: 'I',   romanKo: '1st',  firstFingerSemitone: 2 },
  { id: 'p2',   label: 'II',  romanKo: '2nd',  firstFingerSemitone: 3 },
  { id: 'p3',   label: 'III', romanKo: '3rd',  firstFingerSemitone: 5 },
  { id: 'p4',   label: 'IV',  romanKo: '4th',  firstFingerSemitone: 7 },
];
export const positionById = (id: PositionId) => POSITIONS.find(p => p.id === id)!;

// 반음(크로매틱) 음이름
export const SHARP = ['C','C#','D','D#','E','F','F#','G','G#','A','A#','B'] as const;
export const FLAT  = ['C','Db','D','Eb','E','F','Gb','G','Ab','A','Bb','B'] as const;

export interface NoteName { pc: number; isNatural: boolean; sharp: string; flat: string; }
export function noteNameOf(pc: number): NoteName {
  const i = ((pc % 12) + 12) % 12;
  const sharp = SHARP[i], flat = FLAT[i];
  return { pc: i, isNatural: sharp === flat, sharp, flat };
}

// MIDI / 주파수
export const midiAt = (s: CelloString, semitone: number) => s.midi + semitone;
export const freqOfMidi = (midi: number) => 440 * Math.pow(2, (midi - 69) / 12);
export const freqAt = (s: CelloString, semitone: number) => freqOfMidi(midiAt(s, semitone));

// (포지션, 손가락) → 개방현 위 반음 오프셋 (닫힌 자세: 연속 반음)
export const semitoneFor = (pos: Position, finger: Finger) => pos.firstFingerSemitone + (finger - 1);
// 앞 벌림(forward ext): 1번 고정, 2·3·4 반음 위 (1–2가 온음, 장3도 폭)
export const semitoneForExtended = (pos: Position, finger: Finger) =>
  finger === 1 ? pos.firstFingerSemitone : pos.firstFingerSemitone + (finger - 1) + 1;
// 뒤 벌림(backward ext): 1번만 반음 아래, 나머지는 닫힌 자세
export const semitoneForBackExt = (pos: Position, finger: Finger) =>
  finger === 1 ? pos.firstFingerSemitone - 1 : pos.firstFingerSemitone + (finger - 1);

// (현, 포지션, 손가락) → 음이름 / MIDI
export const noteFor = (s: CelloString, pos: Position, finger: Finger) =>
  noteNameOf(s.pc + semitoneFor(pos, finger));
export const midiFor = (s: CelloString, pos: Position, finger: Finger) =>
  midiAt(s, semitoneFor(pos, finger));

// 음높이(MIDI)로 모든 연주 자리 찾기 (정확 음높이 기준)
export interface Placement { string: StringId; position: PositionId; finger: Finger; semitone: number; midi: number; }
export function placementsForMidi(targetMidi: number): Placement[] {
  const out: Placement[] = [];
  for (const s of STRINGS) for (const p of POSITIONS) for (const f of [1,2,3,4] as Finger[]) {
    const m = midiFor(s, p, f);
    if (m === targetMidi) out.push({ string: s.id, position: p.id, finger: f, semitone: semitoneFor(p,f), midi: m });
  }
  return out;
}
// 피치 클래스(옥타브 무시)로 찾기 — Finder 기본
export function placementsForPc(pc: number): Placement[] {
  const target = ((pc % 12) + 12) % 12;
  const out: Placement[] = [];
  for (const s of STRINGS) for (const p of POSITIONS) for (const f of [1,2,3,4] as Finger[]) {
    const m = midiFor(s, p, f);
    if (((m % 12) + 12) % 12 === target) out.push({ string: s.id, position: p.id, finger: f, semitone: semitoneFor(p,f), midi: m });
  }
  return out;
}
// 추천 운지: 가장 낮은 포지션 우선 (개방현 우선 고려는 호출부에서)
export function recommendedPlacement(ps: Placement[]): Placement | undefined {
  const rank: Record<PositionId, number> = { half:0, p1:1, p2:2, p3:3, p4:4 };
  return [...ps].sort((a,b)=> rank[a.position]-rank[b.position] || a.finger-b.finger)[0];
}

// ---- 한글 음이름 / 계이름 매핑 ----
export const KO_LETTER: Record<string,string> = { C:'다', D:'라', E:'마', F:'바', G:'사', A:'가', B:'나' };
export const SOLFEGE_FIXED: Record<string,string> = { C:'도', D:'레', E:'미', F:'파', G:'솔', A:'라', B:'시' };
const DEGREE_SYL = ['도','도#','레','레#','미','파','파#','솔','솔#','라','라#','시']; // 이동도(올림 표기)
export const solfegeMovable = (pc: number, tonicPc: number) =>
  DEGREE_SYL[((((pc - tonicPc) % 12) + 12) % 12)];

export type LabelSystem = 'en' | 'koLetter' | 'solfegeFixed' | 'solfegeMovable';
export function displayName(
  pc: number, system: LabelSystem,
  opts: { tonicPc?: number; prefer?: 'sharp' | 'flat' } = {}
): string {
  const n = noteNameOf(pc);
  const prefer = opts.prefer ?? 'sharp';
  const pick = prefer === 'sharp' ? n.sharp : n.flat;
  if (system === 'en') return n.isNatural ? n.sharp : pick;
  if (system === 'koLetter') return n.isNatural ? KO_LETTER[n.sharp] : `${KO_LETTER[pick[0]]}${prefer==='sharp'?'♯':'♭'}`;
  if (system === 'solfegeFixed') return n.isNatural ? SOLFEGE_FIXED[n.sharp] : `${SOLFEGE_FIXED[pick[0]]}${prefer==='sharp'?'♯':'♭'}`;
  return solfegeMovable(pc, opts.tonicPc ?? 0); // 'solfegeMovable'
}

// ---- 음계(1포지션 친화 1옥타브). finger 0 = 개방현 ----

export function scaleStepMidi(step: ScaleStep): number {
  const s = stringById(step.string);
  return step.finger === 0 ? s.midi : s.midi + 2 + (step.finger - 1);

// ---- dev 전용 자가 점검 (개발 중 콘솔에 PASS/FAIL) ----
export function selfTest(): { ok: boolean; logs: string[] } {
  const logs: string[] = []; let ok = true;
  const approx = (a:number,b:number)=>Math.abs(a-b)<0.05;
  const fr: Record<StringId,number> = { C:65.41, G:98.00, D:146.83, A:220.00 };
  for (const s of STRINGS){ const f=freqOfMidi(s.midi); const p=approx(f,fr[s.id]); ok = ok && p; logs.push(`${s.sci} ${f.toFixed(2)}Hz ${p?'PASS':'FAIL'}`); }
  const frame = (sid:StringId,pid:PositionId)=>([1,2,3,4] as Finger[]).map(f=>{const n=noteFor(stringById(sid),positionById(pid),f);return n.isNatural?n.sharp:`${n.sharp}/${n.flat}`;});
  const eq=(a:string[],b:string[])=>a.length===b.length&&a.every((x,i)=>x===b[i]);
  const t1=eq(frame('A','half'),['A#/Bb','B','C','C#/Db']); ok = ok && t1; logs.push(`½ A = ${frame('A','half').join(',')} ${t1?'PASS':'FAIL'}`);
  const t2=eq(frame('G','p1'),['A','A#/Bb','B','C']); ok = ok && t2; logs.push(`I G = ${frame('G','p1').join(',')} ${t2?'PASS':'FAIL'}`);
  const t3=eq(frame('G','p4'),['D','D#/Eb','E','F']); ok = ok && t3; logs.push(`IV G = ${frame('G','p4').join(',')} ${t3?'PASS':'FAIL'}`);
  for (const sc of SCALES){ const ms=sc.steps.map(scaleStepMidi); const asc=ms.every((m,i)=>i===0||m>ms[i-1]); ok = ok && asc; logs.push(`${sc.name} asc ${asc?'PASS':'FAIL'}`); }
```

## 6. 음이름 · 계이름 매핑 표 (라벨 시스템 기준)

| 영문 음이름 | 한글 음이름(고정) | 계이름(고정도, 도=C) | 피치클래스(pc) |
|:---:|:---:|:---:|:---:|
| C | 다 | 도 | 0 |
| D | 라 | 레 | 2 |
| E | 마 | 미 | 4 |
| F | 바 | 파 | 5 |
| G | 사 | 솔 | 7 |
| A | 가 | 라 | 9 |
| B | 나 | 시 | 11 |

- **올림(♯)/내림(♭)** : 같은 음높이의 다른 표기(이명동음). 예) `C♯ = D♭ = 다♯ = 도♯`.
- **고정도(Fixed-do)** : 도 = C 고정(입문 친화). **이동도(Movable-do)** : 도 = 그 조의 으뜸음(예: 사장조면 도 = G).
- 개방현을 계이름으로(고정도): C=도, G=솔, D=레, A=라.

### (참고) 검증된 1포지션 음계 운지
| 음계 | 진행(현·손가락) | 음이름 |
|---|---|---|
| **다장조 C** | C0·C1·C3·C4 → G0·G1·G3·G4 | C D E F G A B C |
| **사장조 G** | G0·G1·G3·G4 → D0·D1·D3·D4 | G A B C D E F♯ G |
| **라장조 D** | D0·D1·D3·D4 → A0·A1·A3·A4 | D E F♯ G A B C♯ D |

> 공통 패턴(각 현): **0·1·3·4** (2번 손가락은 온음 때문에 건너뜀). 이것이 첼로 닫힌 자세의 핵심 감각이다.

## 9. 정확성 가드레일 & 흔한 실수 (막히면 이 블록을 다시 붙여넣기)

```text
[정확성 점검]
- 첼로는 C·G·D·A 완전5도 조율. 연주자 시점 왼→오른쪽 C·G·D·A. (관객 시점은 거울상)
- 닫힌 자세(closed)에서 손가락 1·2·3·4는 연속 "반음" 간격(1~4가 단3도 폭).
  온음은 손가락을 건너뛰거나(예: 1·3) 익스텐션으로 낸다. ← 바이올린식 "온음 기본"과 다름!
- 포지션의 1번 손가락 위치(개방현 위 반음): ½=+1, I=+2, II=+3, III=+5, IV=+7.
- 앞 벌림: 1번 고정, 2·3·4를 반음 위로(1–2가 온음, 장3도 폭). 뒤 벌림: 1번만 반음 아래.
- ♯와 ♭는 같은 음높이(이명동음). UI에선 둘 다 보여주거나 설정으로 선택.
- 주파수는 freq = 440 * 2^((midi-69)/12). 개방현 MIDI: C2=36, G2=43, D3=50, A3=57.
- 위 사실과 어긋나는 코드가 있으면 멈추고 celloData.ts 기준으로 고쳐라.

[흔한 실수]
- 음 자리를 데이터가 아니라 화면에 직접 하드코딩 → 금지. 항상 celloData로 계산.
- 오디오를 사용자 제스처 전에 시작 → 브라우저가 막음. 첫 탭에서 Tone.start().
- 1포지션 D현을 D·E·F·G(온음 위주)로 표기 → 틀림. 닫힌 프레임은 E·F·F#·G(반음).
- 저작권 있는 첼로 샘플/유행가 멜로디 사용 → 금지(합성음 + 퍼블릭 도메인만).
- 색만으로 현/정오답 구분 → 색맹 사용자 배제. 색+모양+텍스트 병행.
- 첼로에 "프렛"이라는 표현 사용 → 금지. "손가락 자리", "위치 표시선"으로.
- 모바일에서 호버 의존 / 작은 터치 타깃 / 상단에 주요 버튼 → 금지. 탭 기반 + ≥48px + 엄지존.
```
