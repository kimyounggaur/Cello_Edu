# 첼로 운지법·계이름 연주 웹앱 개발용 바이브코딩 프롬프트

> 목적: 초보 첼리스트가 **현 → 손가락 → 음이름 → 계이름 → 실제 소리**를 한 화면에서 연결해 익히는 웹앱을 만들기 위한, 복사해서 코딩 에이전트에 넣을 수 있는 단계별 프롬프트 설계서입니다.  
> 추천 사용처: Cursor, Claude Code, Codex, Windsurf, Replit Agent, v0, Bolt 등.  
> 기본 언어: 한국어 UI. 기본 계이름 체계: **고정도법 C=도**. 기본 조율: **C2-G2-D3-A3 / 완전5도**.

---

## 0. 이 파일을 쓰는 방법

1. 먼저 **1장 “마스터 프롬프트”** 전체를 코딩 에이전트에 붙여 넣습니다.
2. 에이전트가 전체를 한 번에 구현하려 하면 너무 넓어질 수 있으므로, 실제 개발은 **8장 “단계별 실행 프롬프트”**를 1단계부터 차례대로 넣는 방식을 권장합니다.
3. 각 단계가 끝나면 **완료 기준**과 **검수 질문**을 확인합니다.
4. 운지 데이터는 절대 감으로 작성하지 말고, `/src/lib/music/` 아래의 데이터 파일과 테스트로 검증하게 합니다.
5. 첼로 운지는 문맥, 음계, 포지션, 손 크기, 레퍼토리에 따라 달라질 수 있으므로, 앱에서는 “유일한 정답”이 아니라 **초급 표준 추천 운지 + 대체 운지** 구조로 설계합니다.

## 1. 마스터 프롬프트: 전체 앱을 설계하고 구현해줘

아래 블록을 그대로 복사해서 코딩 에이전트에 입력하세요.

````prompt
너는 시니어 프론트엔드 개발자, 음악 교육용 UX 디자이너, 첼로 기초 이론을 이해하는 교육 앱 설계자다.

목표는 한국어 사용자가 첼로의 운지법과 계이름 연주 방법을 가장 직관적으로 배울 수 있는 웹앱을 만드는 것이다. 정적인 운지표가 아니라, 사용자가 직접 누르고 듣고 따라 치면서 익히는 인터랙티브 학습 앱을 만들어라.

핵심 학습 연결은 다음 순서다.
1. 현 이름을 본다: C현, G현, D현, A현
2. 손가락 번호를 본다: 0=open, 1=검지, 2=중지, 3=약지, 4=새끼, T=엄지 포지션
3. 음이름을 본다: C, D, E, F, G, A, B + #/b
4. 한국어 계이름을 본다: 도, 레, 미, 파, 솔, 라, 시
5. 실제 음을 듣는다.
6. 가상 첼로 지판에서 같은 위치를 다시 찾아 누른다.
7. 짧은 멜로디를 계이름으로 읽고 운지로 연주한다.

기술 스택은 다음을 기본으로 한다.
- Next.js App Router + TypeScript + React
- Tailwind CSS
- 클라이언트 오디오: Web Audio API 우선, 필요하면 Tone.js를 보조적으로 사용
- 악보 렌더링은 MVP에서는 간단한 SVG 오선 표시로 시작하고, 확장 단계에서 VexFlow를 선택적으로 사용
- 백엔드 없이 MVP를 완성한다. 진도 저장은 localStorage 또는 IndexedDB를 사용한다.
- 모든 음악 데이터는 하드코딩된 UI 안에 섞지 말고 `/src/lib/music/`에 순수 데이터와 순수 함수로 분리한다.

반드시 지켜야 할 음악 기준:
- 표준 첼로 조율은 낮은 현부터 C2, G2, D3, A3다.
- 인접 현은 완전5도 간격이다.
- MIDI 번호는 C2=36, G2=43, D3=50, A3=57, A4=69를 기준으로 한다.
- 주파수 공식은 `frequency = 440 * 2 ** ((midi - 69) / 12)`를 사용한다.
- 기본 계이름은 고정도법으로 C=도, D=레, E=미, F=파, G=솔, A=라, B=시다.
- 처음에는 초보자용 1포지션을 가장 정확하고 풍부하게 구현한다.
- 확장 운지는 “뒤확장/낮은 1번”, “앞확장”으로 구분한다.
- 포지션 5 이상과 엄지 포지션은 MVP에서는 요약/미리보기로 제공하고, 정확한 운지 데이터는 교사 검수 가능한 JSON 구조로 확장 가능하게 만든다.

사용자 경험 원칙:
- 첫 화면에서 설명을 많이 읽게 하지 말고, 사용자가 바로 A현 open, D현 open 등을 눌러 소리를 들을 수 있어야 한다.
- 모든 노트 마커는 색, 손가락 번호, 음이름, 계이름을 동시에 제공해야 한다.
- 초보자는 “현 이름”보다 “어디를 누르는지”가 먼저 보이므로, 가상 지판 위의 위치가 가장 크고 명확해야 한다.
- 첼로는 프렛이 없으므로 UI에는 “프렛”이라는 표현을 쓰지 말고 “위치 표시선”, “반음 위치”, “손가락 자리”라고 표현한다.
- 모바일에서 엄지로 누르기 쉬워야 하므로 터치 타깃은 최소 44px 이상이어야 한다.
- 색만으로 정보를 전달하지 말고 텍스트, 아이콘, aria-label을 함께 제공한다.
- 모든 오디오 재생은 브라우저 정책을 고려해 사용자의 첫 탭 이후 AudioContext를 시작한다.

앱 구조:
- `/` : 오늘의 5분 학습 대시보드
- `/learn/fingerboard` : 인터랙티브 지판 학습
- `/learn/solfege` : 계이름/음이름 연결 학습
- `/practice/find-note` : “이 음을 찾아 눌러보세요” 퀴즈
- `/practice/fingering` : “이 운지의 음이름은?” 퀴즈
- `/practice/melody` : 계이름 멜로디를 가상 첼로로 연주
- `/library/chart` : 전체 운지표 라이브러리
- `/settings` : 표기 설정, #/b 우선 표기, 고정도/이동도 토글 예정, 왼손잡이/오른손잡이 지판 방향 토글

주요 컴포넌트:
- `VirtualCelloFingerboard`
- `StringRail`
- `NoteMarker`
- `FingeringCard`
- `SolfegeBadge`
- `AudioPlayButton`
- `MiniStaff`
- `LessonStepper`
- `QuizEngine`
- `ProgressPanel`
- `SettingsPanel`

핵심 기능:
1. 가상 첼로 지판
   - C, G, D, A 네 현을 표시한다.
   - 현마다 open note와 1포지션 정상 운지, 뒤확장, 앞확장을 표시한다.
   - 노트 마커를 누르면 카드가 열린다.
   - 카드에는 현, 손가락, 포지션, 음이름, 계이름, MIDI, 주파수, 짧은 설명, 재생 버튼이 보인다.

2. 운지-계이름 카드
   - 예: “A현 1번 손가락 = B3 = 시”
   - 예: “D현 2번 손가락 = F3 = 파”
   - 예: “G현 4번 손가락 = C3 = 도”
   - 사용자가 같은 음을 다른 현에서 낼 수 있는 경우 대체 운지를 보여준다.

3. 소리 재생
   - 클릭한 음을 Web Audio API로 재생한다.
   - 기본 파형은 너무 날카롭지 않게 sine 또는 triangle에서 시작한다.
   - 짧은 attack/release envelope를 적용해 클릭 노이즈를 줄인다.
   - 나중에 오픈 라이선스 첼로 샘플을 붙일 수 있도록 `AudioEngine` 인터페이스를 추상화한다.

4. 계이름 멜로디 연주
   - 앱이 “도 레 미 파 솔” 같은 계이름 시퀀스를 보여준다.
   - 사용자는 지판에서 해당 음들을 순서대로 누른다.
   - 정답이면 다음 음으로 넘어가고, 틀리면 “현은 맞았지만 손가락이 달라요”처럼 구체적 피드백을 준다.
   - C장조, G장조, D장조 초급 패턴을 제공한다.

5. 퀴즈 모드
   - 음 찾기: “D현 F#을 눌러보세요.”
   - 계이름 찾기: “솔을 눌러보세요.”
   - 운지 읽기: “A현 2번 손가락은 어떤 계이름인가요?”
   - 듣고 찾기: 음을 듣고 지판에서 찾는다.
   - 실수 패턴을 기록해 다음 문제를 조정한다.

6. 진도 저장
   - 완료한 레슨, 정답률, 최근 틀린 음, 즐겨찾기한 음을 localStorage에 저장한다.
   - 개인정보 수집 없이 로컬 저장만 사용한다.

7. 검수와 테스트
   - `src/lib/music/music.test.ts`를 만들고 open string MIDI, 주파수 공식, 계이름 매핑, 1포지션 매핑을 테스트한다.
   - 잘못된 운지를 UI에 표시하지 않도록 데이터 스키마를 엄격히 만든다.
   - `teacher-review.md` 파일을 만들어 교사가 검수해야 할 포인트를 자동으로 정리한다.

디자인 방향:
- 분위기: 따뜻하고 명확한 음악 교실 느낌.
- 색은 현별로 구분하되, 색만으로 의미를 전달하지 않는다.
- 큰 카드, 큰 버튼, 짧은 문장, 즉각적인 소리 피드백.
- “외우세요”가 아니라 “눌러보고 들어보세요” 중심.
- 한국어 용어를 우선 사용하되 음이름은 국제 표기 C/D/E도 같이 보여준다.
````

## 2. 제품 정의: “가장 직관적인 첼로 학습 앱”의 기준

### 2.1 핵심 사용자

- 첼로를 처음 배우는 초보자
- 악보의 음표와 실제 첼로 지판 위치가 연결되지 않는 학습자
- 계이름은 아는데 첼로에서 어디를 눌러야 하는지 모르는 학습자
- 레슨 사이에 혼자 복습하고 싶은 학생
- 어린 학습자를 지도하는 보호자 또는 교사

### 2.2 앱의 핵심 한 문장

> “화면의 첼로를 누르면 그 자리의 손가락, 음이름, 계이름, 소리가 동시에 연결되는 학습 앱.”

### 2.3 초보자가 반드시 해결해야 하는 혼란

| 초보자의 혼란 | 앱이 제공해야 할 해결 |
|---|---|
| C현/G현/D현/A현 위치를 헷갈림 | 현 이름을 항상 고정 표시하고, 현별 색과 라벨을 함께 제공 |
| 1번 손가락이 어느 음인지 모름 | “A현 1번 = B = 시”처럼 현+손가락+음이름+계이름을 한 카드로 표시 |
| 같은 계이름이 여러 위치에 있어 헷갈림 | “초급 추천 위치”와 “대체 위치”를 분리해서 표시 |
| 음이름 C와 계이름 도의 관계를 헷갈림 | 고정도법 기본값: C=도, D=레, E=미, F=파, G=솔, A=라, B=시 |
| 악보와 지판이 연결되지 않음 | 작은 오선 악보 + 지판 마커 + 소리 재생을 동시에 제공 |
| 소리를 듣지 않고 외움 | 모든 학습 카드에 “듣기” 버튼 제공 |

## 3. 정보 구조와 화면 설계

### 3.1 전체 내비게이션

```text
홈
├─ 오늘의 5분
├─ 지판으로 배우기
├─ 계이름으로 연주하기
├─ 퀴즈
│  ├─ 음 찾기
│  ├─ 운지 맞히기
│  ├─ 듣고 찾기
│  └─ 계이름 순서 연주
├─ 운지표 라이브러리
└─ 설정
```

### 3.2 홈 화면

홈 화면은 설명서가 아니라 “시작 버튼”이어야 합니다.

필수 UI:

- 큰 제목: `첼로 운지와 계이름을 눌러서 배워요`
- 오늘의 추천 버튼 3개
  - `A현 open부터 듣기`
  - `1포지션 5분 연습`
  - `도레미 멜로디 연주`
- 최근 틀린 음 3개
- 오늘의 진도: 0/5분, 정답률, 완료한 카드

### 3.3 지판 학습 화면

가장 중요한 화면입니다.

배치:

```text
[상단] 학습 단계 / 설정 토글 / 듣기 볼륨
[중앙] 대형 가상 첼로 지판 SVG
[하단] 선택한 음 상세 카드
[오른쪽 또는 하단] 미니 오선 + 계이름 설명
```

지판 방향 옵션:

- `세로 보기`: 실제 첼로 느낌. 위쪽이 높은 포지션, 아래쪽이 너트.
- `가로 보기`: 모바일과 태블릿에서 넓게 보기 좋음.
- `초보자 보기`: 현을 A-D-G-C 순서 또는 C-G-D-A 순서로 바꾸는 토글 제공.

마커 표시 규칙:

```text
[손가락 번호]
음이름
계이름
```

예:

```text
1
B
시
```

### 3.4 계이름 연주 화면

목표는 “도레미를 보면 손이 어디로 가야 하는지”를 익히는 것입니다.

필수 요소:

- 현재 멜로디: `도 - 레 - 미 - 파 - 솔`
- 현재 눌러야 할 음 강조
- 추천 현/손가락 안내
- 사용자가 지판을 누르면 정답/오답 피드백
- 정답이면 다음 음으로 자동 이동
- 느리게 듣기, 한 음씩 듣기, 전체 듣기 버튼

예시 피드백:

- 정답: `좋아요! G현 4번 손가락 C3 = 도예요.`
- 현은 맞고 손가락만 틀림: `G현은 맞아요. 이번 음은 4번 손가락 자리예요.`
- 계이름은 맞지만 옥타브가 다름: `도는 맞지만 이번 멜로디에서는 C3 도를 사용해요.`
- 완전히 틀림: `다시 들어보세요. 목표는 레, 추천 위치는 C현 1번 손가락 D2예요.`

### 4.6 음정 계산 함수

```ts
export function midiToFrequency(midi: number): number {
  return 440 * Math.pow(2, (midi - 69) / 12);
}

export function midiToPitchClass(midi: number): number {
  return ((midi % 12) + 12) % 12;
}
```

### 4.7 지판 위치 계산 함수

실제 악기처럼 반음 간격이 위로 갈수록 좁아지는 느낌을 주기 위한 시각화 공식입니다.

```ts
export function semitoneToFingerboardRatio(semitoneFromOpen: number): number {
  // 0이면 너트 근처, 숫자가 커질수록 브리지 방향.
  // 이 값은 실제 물리 길이의 비율을 시각화하기 위한 값이다.
  return 1 - 1 / Math.pow(2, semitoneFromOpen / 12);
}
```

## 5. UX 세부 사양

### 5.1 노트 마커 디자인

각 노트 마커는 다음 정보를 포함합니다.

```text
┌─────────┐
│   1     │ ← 손가락 번호
│   B     │ ← 음이름
│   시    │ ← 계이름
└─────────┘
```

상태:

- 기본: 작은 원형/캡슐
- hover/focus: 살짝 커짐, 음이름 강조
- selected: 상세 카드와 연결선 표시
- correct: 체크 애니메이션
- wrong: 부드러운 흔들림 + 정확한 힌트
- disabled: 현재 레슨 범위 밖의 음은 흐리게 표시

### 5.2 상세 카드 내용

```text
A현 1번 손가락
B3 / 시

포지션: 1포지션 정상 운지
주파수: 246.94 Hz
설명: A현에서 1번 손가락을 누르면 B3, 고정도법 계이름으로 ‘시’입니다.

[소리 듣기] [같은 계이름 다른 위치 보기] [퀴즈로 연습]
```

### 5.3 같은 음/같은 계이름 보기

두 종류를 분리해야 혼란이 줄어듭니다.

- **같은 실제 음**: MIDI가 같은 음. 예: D4는 A현 4번 또는 D현의 높은 포지션에서 가능.
- **같은 계이름**: 옥타브는 달라도 pitch class가 같은 음. 예: C2, C3, C4는 모두 도.

UI 라벨:

```text
같은 실제 음
같은 계이름
```

### 8.2 2단계: 음악 데이터 모델 만들기

````prompt

- C2=36, G2=43, D3=50, A3=57
- A4=69, A4=440Hz
- 주파수 공식: 440 * 2 ** ((midi - 69) / 12)
- 고정도법: C=도, D=레, E=미, F=파, G=솔, A=라, B=시

- 1포지션 forward extension: 1=+2, 2x=+4, 3x=+5, 4x=+6

- A4 MIDI 69의 주파수는 440Hz
- A현 open은 A3, MIDI 57, 계이름 라
- D현 1번 normal은 E3, 계이름 미
- G현 4번 normal은 C3, 계이름 도
- A현 2번 normal은 C4, 계이름 도
- D현 low-1은 Eb3/D#3 계열

주의:
- UI 컴포넌트 안에 음악 데이터를 직접 하드코딩하지 말 것.
- enharmonic 표기는 sharp/flat 둘 다 지원할 것.
- 테스트가 실패하면 UI를 만들지 말고 데이터부터 수정할 것.

완료 후 보고:
- 데이터 구조
- 테스트 실행 방법
- 통과한 테스트 목록
````

## 16. 개발자가 놓치기 쉬운 핵심 주의사항

1. **오디오를 서버 컴포넌트에서 만들지 말 것**  
   AudioContext는 브라우저 API이므로 클라이언트 컴포넌트에서 사용자 제스처 이후 생성해야 합니다.

2. **첼로 지판을 기타 프렛보드처럼 만들지 말 것**  
   반음 위치 표시선은 학습용 보조선일 뿐입니다. “프렛”이라는 표현을 사용하면 안 됩니다.

3. **계이름과 음이름을 분리하되 항상 연결해서 보여줄 것**  
   C는 음이름, 도는 계이름입니다. 초보자에게는 둘을 동시에 보여주는 편이 가장 직관적입니다.

4. **운지는 유일한 정답이 아님**  
   앱의 정답 판정은 초급 레슨 범위에서는 엄격하게 하되, 라이브러리 화면에서는 대체 운지를 보여줄 수 있어야 합니다.

5. **확장 운지를 normal에 섞지 말 것**  
   뒤확장과 앞확장은 별도 배지와 별도 데이터로 처리해야 합니다.

6. **색상만 믿지 말 것**  
   색각 이상 사용자를 위해 현 이름, 손가락 번호, 텍스트 라벨을 항상 함께 표시합니다.

7. **처음부터 고급 포지션을 과하게 넣지 말 것**  
   학습 앱의 신뢰도는 1포지션의 정확성과 사용성에서 결정됩니다. 5~7포지션과 엄지 포지션은 검수 가능한 확장 구조로 남겨두는 편이 안전합니다.

## 17. 최종 성공 기준

이 웹앱의 성공 기준은 기능 개수가 아니라 학습자가 다음 문장을 말할 수 있는지입니다.

> “A현 1번 손가락을 누르면 B, 계이름 시가 나고, 이 소리가 바로 그 음이구나.”

최소 성공 시나리오:

1. 사용자가 첫 화면에서 `A현 open`을 누른다.
2. 앱이 A3 / 라를 보여주고 소리를 들려준다.
3. 사용자가 `A현 1번`을 누른다.
4. 앱이 B3 / 시를 보여주고 소리를 들려준다.
5. 사용자가 계이름 멜로디 `라 시 도 레`를 보고 지판에서 순서대로 누른다.
6. 앱이 구체적으로 칭찬하거나 교정한다.
7. 사용자는 “현, 손가락, 계이름, 소리”가 연결되었다고 느낀다.
